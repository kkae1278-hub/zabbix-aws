## 1. システム概要

### 目的

AWS 上に Zabbix 7.x をベースとした監視基盤を構築し、EC2 インスタンスのリソース監視・アラート通知を実現する。

### 実現する機能

| 機能 | 内容 |
|---|---|
| インフラ監視 | CPU・メモリ・ディスク・ネットワーク（Zabbix Agent + CloudWatch Agent） |
| HTTP 監視 | ALB 経由の Web シナリオ監視（200/404/503 ステータス確認） |
| アラート通知 | CloudWatch Alarm → SNS メール通知 |
| ダッシュボード | Zabbix Web UI + CloudWatch Dashboard |
| セキュアアクセス | SSM Session Manager（SSH・踏み台サーバー不要） |
| ログ管理 | CloudWatch Logs へのログ転送 |
| IaC 管理 | Terraform による全リソース管理・S3 Remote State |

### ネットワーク設計

| VPC | サブネット | CIDR | 配置リソース |
|---|---|---|---|
| Zabbix VPC | Public Subnet ×2 | 10.0.1.0/24, 10.0.2.0/24 | IGW（初回 NAT Gateway） |
| Zabbix VPC | Private Subnet ×2 | 10.0.10.0/24, 10.0.11.0/24 | Zabbix Server, RDS |
| Monitored VPC | Private Subnet ×2 | 10.1.10.0/24, 10.1.11.0/24 | AP Server ×2, Internal ALB |

### セキュリティグループ設計

```
[Zabbix Server SG]
  ├─ Ingress: 10051/tcp from Monitored VPC（Active check trapper）
  └─ Egress:  All

[RDS SG]
  └─ Ingress: 3306/tcp from Zabbix Server SG

[Monitored ALB SG]
  ├─ Ingress: 80/tcp from 0.0.0.0/0（Internal ALB）
  └─ Egress:  All

[Zabbix Agent SG]
  ├─ Ingress: 10050/tcp from Zabbix VPC（Passive check）
  ├─ Ingress: 80/tcp from Monitored ALB SG
  └─ Egress:  All
```

## 4. 前提条件・事前準備

### 必要ツール

| ツール | バージョン | 用途 |
|---|---|---|
| Terraform | >= 1.6.0 | IaC によるリソース管理 |
| AWS CLI | v2 | AWS 操作 |
| Session Manager Plugin | 最新版 | EC2 への SSM 接続 |

### インストール（Windows）

```powershell
# Terraform
winget install HashiCorp.Terraform
terraform --version  # v1.6.0 以上

# AWS CLI v2
# https://awscli.amazonaws.com/AWSCLIV2.msi をダウンロード・実行
aws --version        # aws-cli/2.x 以上

# Session Manager Plugin
# https://s3.amazonaws.com/session-manager-downloads/plugin/latest/windows/SessionManagerPluginSetup.exe
session-manager-plugin --version
```

### AWS 認証（IAM Identity Center / SSO）

```powershell
aws configure sso
# SSO Start URL と Region を入力

aws sso login --profile zabbix-dev
$env:AWS_PROFILE = "zabbix-dev"

# 確認
aws sts get-caller-identity
```

## 5. 構築手順

### Step 1: Remote State 用インフラの作成（初回のみ）

```powershell
cd terraform/bootstrap
terraform init
terraform apply
# 出力の backend_config を控えておく
```

`terraform/main.tf` の backend ブロックに出力値を貼り付け、State を移行する。

```powershell
cd ../
terraform init -migrate-state
```

### Step 2: 初回のみ NAT Gateway を追加してデプロイ

> Zabbix Server のセットアップには `repo.zabbix.com` への通信が必要。  
> NAT Gateway を一時追加し、セットアップ完了後に削除する。  
> 詳細は [docs/troubleshooting.md](troubleshooting.md) を参照。

```powershell
cd terraform

# NAT Gateway を有効化した状態で apply
terraform init
terraform plan
terraform apply
```

**所要時間**: 約 10〜15 分（RDS 作成が最も時間がかかる）

```powershell
# デプロイ後にリソース情報を確認
terraform output
```

### Step 3: Zabbix のセットアップ完了を確認

```powershell
# Zabbix Server に SSM 接続
aws ssm start-session --target (terraform output -raw zabbix_server_instance_id)

# セットアップログを確認
sudo tail -f /var/log/zabbix-setup.log
# "=== Zabbix Setup Complete ===" が表示されたら完了
```

### Step 4: NAT Gateway を削除（コスト削減）

セットアップ完了後、NAT Gateway リソースを `vpc.tf` から削除して `terraform apply`。

### Step 5: Zabbix GUI にアクセス（SSM Port Forwarding）

```powershell
aws ssm start-session `
  --target (terraform output -raw zabbix_server_instance_id) `
  --document-name AWS-StartPortForwardingSession `
  --parameters '{\"portNumber\":[\"80\"],\"localPortNumber\":[\"8080\"]}'
```

ブラウザで `http://localhost:8080/zabbix` を開く。

- Username: `Admin`
- Password: `zabbix`（**初回ログイン後に必ず変更する**）

### Step 6: Admin パスワードの変更

**GUI**: 右上アイコン → User settings → Change password

**または SSM 経由でスクリプト実行**（`terraform/scripts/change_zabbix_password.sh`）:

```bash
# Zabbix Server の SSM セッション内で実行
sudo bash /tmp/change_zabbix_password.sh 'NewPassword123!'
```

### Step 7: 監視設定

詳細は [docs/zabbix_monitoring_setup.md](zabbix_monitoring_setup.md) を参照。

1. ALB HTTP 監視（Web シナリオ）
2. AP サーバー Agent 監視（ホスト登録）
3. ホスト自動登録（Auto-registration Action）
4. アラートアクション（SNS 通知）

## 6. 動作確認

### インフラ確認

```powershell
# RDS 状態
aws rds describe-db-instances `
  --db-instance-identifier zabbix-db `
  --query "DBInstances[*].DBInstanceStatus" `
  --region ap-northeast-1

# ALB ターゲットヘルスチェック
aws elbv2 describe-target-health `
  --target-group-arn (aws elbv2 describe-target-groups --names zabbix-monitored-tg --query "TargetGroups[0].TargetGroupArn" --output text --region ap-northeast-1) `
  --region ap-northeast-1
```

### Web シナリオ確認

```powershell
$ALB = "internal-zabbix-monitored-alb-xxxxxxxxx.ap-northeast-1.elb.amazonaws.com"
# ※ ALB は内部向けのため Zabbix Server から実行すること
```

### チェックリスト

```

  [ ] terraform apply 正常完了（エラーなし）
  [ ] RDS ステータス: available
  [ ] ALB ターゲットヘルスチェック: healthy

Zabbix
  [ ] Web UI にアクセス可能（SSM Port Forwarding）
  [ ] Admin パスワード変更済み
  [ ] Web シナリオ（ALB HTTP Check）が OK
  [ ] AP サーバーのホストが ZBX 緑表示
  [ ] グラフ（CPU・メモリ）がデータを表示

CloudWatch
  [ ] Dashboard にメトリクスが表示（5 行 ×2 列）
  [ ] CWAgent でメモリ・ディスクが収集されている
  [ ] CloudWatch Alarms が OK 状態

通知
  [ ] SNS サブスクリプションが Confirmed
```

## 7. コスト管理

### 概算コスト（ap-northeast-1）

| リソース | 仕様 | 1日 | 1週間 |
|---|---|---|---|
| EC2 t3.micro × 3 | Zabbix Server + AP ×2 | $0.98 | $6.85 |
| RDS db.t3.micro | MySQL 8.0 / 20GB gp3 | $0.71 | $4.97 |
| **VPC Interface Endpoint × 8** | **SSM 系 / Secrets / Logs** | **$5.38** | **$37.63** |
| ALB (Internal) | - | $0.59 | $4.13 |
| EBS 30GB gp3 × 3 | EC2 ルートボリューム | $0.29 | $2.02 |
| CloudWatch / S3 / その他 | - | $0.13 | $0.91 |
| **合計** | | **≈ $8.08** | **≈ $56.51** |

> **最大コスト要因は VPC Interface Endpoint（全体の約 66%）。**

### コスト削減操作

```powershell
# 就寝前: VPC Endpoint のみ削除（$5.38/日 節約）
terraform destroy -target="aws_vpc_endpoint.ssm" -target="aws_vpc_endpoint.ssmmessages" -target="aws_vpc_endpoint.ec2messages" -target="aws_vpc_endpoint.secretsmanager" -target="aws_vpc_endpoint.logs" -target="aws_vpc_endpoint.monitored_ssm" -target="aws_vpc_endpoint.monitored_ssmmessages" -target="aws_vpc_endpoint.monitored_ec2messages"

# 翌朝: 全リソース復元
terraform apply
```

```powershell
cd terraform

```

>   --secret-id "zabbix/rds/password" `
>   --force-delete-without-recovery `
>   --region ap-northeast-1
> ```

## 9. トラブルシューティング

詳細は [docs/troubleshooting.md](troubleshooting.md) を参照。

| 問題 | 原因 | 対応 |
|---|---|---|
| Zabbix インストール失敗 | AL2023 と el9 パッケージ非互換 | Ubuntu 22.04 を使用 |
| 外部リポジトリに接続できない | NAT Gateway なし | 初回のみ一時追加 |
| dnf タイムアウト | S3 VPC Endpoint なし | S3 Gateway Endpoint を追加 |
| user_data が途中終了 | `set -e` による即時終了 | `\|\| true` を付ける |
| bash 変数が Terraform に解釈される | `${}` の衝突 | `$${}` でエスケープ |
| IMDS 取得失敗 | IMDSv2 未対応の curl | Token ヘッダーを付与 |
| サービス起動でブロック | systemd PID 待ち | `--no-block` を使用 |
| JSON パラメータが壊れる | PowerShell のクォート除去 | `\"` でエスケープ |
| RDS インスタンス作成失敗 | db.t3.micro が Performance Insights 非対応 | `performance_insights_enabled = false` |
| ALB ログが Access Denied | S3 バケットポリシー適用タイミング | `depends_on` を追加 |
| Ubuntu で apt が失敗 | Ubuntu リポジトリは S3 外のため VPC Endpoint 非対応 | 監視対象は AL2023 を使用 |
